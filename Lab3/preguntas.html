CSS:

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS

    • Se usa !important en una regla de estilo para dar prioridad a una instrucción de forma que las instrucciones siguientes no puedan sobrescribirla y la especificada como !important predomine.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Debemos considerar que colocar una imagen como fondo de nuestra pagina puede dificultar mucho la presentación de la información de manera clara, por eso es importante tener en cuenta las distintas definiciones de pantalla, el tamaño de la imagen, su resolución, que permita la lectura del contenido, etc…

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Unidades como % (porcentaje) son relativas, o sea que hace referencia a otra medida, esta unidad se puede usar para el layout del documento.
    • El px (pixel) es relativa respecto de la resolución de la pantalla es muy usado pues permite que determinados elementos se visualicen con tamaños bastante precisos.
    • En el caso de pt (punto) es una unidad absoluta, que se determinan de forma concretas, el punto se usa a veces para definir el tamaño de letra.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • El método POST, este puede enviar datos al servidor y crear un recurso.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • El método GET, este puede leer los datos del URL y recuperarlos.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • No hay error, es un código de respuesta de estado satisfactorio, indica que la solicitud ha tenido éxito.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Ahorra todo el trafico generado por los espacios y saltos que hay cuando escribimos un documento css.
