Programación orientada a eventos:

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    • JavaScript proporciona numerosas formas de validación de los datos de un formulario en el navegador antes de ser enviado al servidor, de esta manera informacion que llega al servidor tiene menos porcentaje de error y se ahorra el largo proceso de que el servidor envie de nuevo la información al navegador.

  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    • Usando el atributo novalidate.

  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    • Es una forma de que el usuario se de cuenta de los errores que pudo cometer tal vez por accidente y que la correccion no siempre dependa la respuesta del servidor.
